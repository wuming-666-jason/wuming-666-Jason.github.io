Linux 命令行核心命令汇总

类别 常用命令 功能说明
系统信息 whoami, hostname, uname -a, lsb_release 查看用户、主机、系统版本信息
文件操作 ls, cd, mkdir, touch, cp, mv, rm 文件和目录的增删改查
资源监控 free -h, df -h, top, du -sh 查看内存、磁盘、进程和目录大小
用户管理 adduser, passwd, usermod, deluser, su 用户的创建、修改、切换和删除
权限管理 chmod, chown, chgrp 文件的读/写/执行权限与所有权管理
环境变量 env, echo $VAR, export, unset, source 查看、设置、删除和应用环境变量

关键配置文件对比

配置文件 作用范围 加载时机 典型用途
~/.bashrc 当前用户 每次打开新终端 用户别名（alias）、自定义 PATH、个人环境变量
~/.profile 当前用户 用户登录时执行一次 登录时初始化配置
/etc/environment 所有用户 系统启动时（需重启） 系统级环境变量（不使用 export 语法）
/etc/profile 所有用户 用户登录时执行 全局 Shell 初始化，遍历加载 profile.d
/etc/profile.d/*.sh 所有用户 由 /etc/profile 加载 模块化系统级配置（推荐方式）
/etc/passwd 所有用户 系统查询 存储用户账户基本信息
/etc/group 所有用户 系统查询 存储用户组信息
/etc/shadow 仅 root 系统查询 存储用户加密密码（不可明文查看）

环境变量设置方法对比

方法 生效范围 有效期 语法
命令行 export 当前终端 终端关闭后失效 export VAR=value
写入 ~/.bashrc 当前用户 永久 export VAR=value
写入 /etc/environment 所有用户 永久（需重启） VAR=value（不用 export）

写入 /etc/profile.d/*.sh 所有用户 永久（source 或重启） export VAR=value（推荐）

6、出现问题及解决方案

问题一：adduser 命令的 GECOS 字段困惑

现象：执行 sudo adduser testuser 创建新用户时，输入密码后系统依次弹出以下交互提示：

看到这些提示时不知道如何填写，担心填错会导致用户创建失败或影响后续使用。

原因分析：这些字段被称为 GECOS（General Electric Comprehensive Operating System）字段，是 Unix 早期遗留的用户信息字段，主要用于大型企业/学校环境中记录用户的真实身份信息（如办公室位置、联系电话等）。对于个人实验环境来说，这些信息没有任何实际用途。

解决方法：全部字段直接按回车键跳过（留空），最后系统会提示确认：

输入 Y 确认即可。用户创建成功不受影响。即便已填写，也可随时通过 sudo chfn testuser 或 sudo usermod 修改这些信息。

经验总结：Linux 命令行中很多交互式提示的可选字段都标注了 []，括号内为空表示留空是可接受的，按回车跳过即可。

问题二：deluser -r 参数歧义错误

现象：执行 sudo deluser -r testuser 删除用户时，命令报错：

命令未能执行，提示 -r 参数匹配到了两个选项。

原因分析：deluser 命令有两个以 r 开头的长选项——--remove-home（删除用户主目录和邮箱）和 --remove-all-files（删除用户拥有的所有文件）。使用短参数 -r 时，系统无法判断你想要的是哪一个，因此拒绝执行并给出提示。

解决方法：使用完整的参数名替代缩写：

即可成功删除用户及其主目录。如果需要删除该用户拥有的全部文件（更彻底），则应使用 --remove-all-files。

经验总结：Linux 命令中，短参数（单个字母）在多个长选项以同一字母开头时会失效。建议养成使用完整参数名的习惯，既避免歧义，也提高命令可读性。可以通过 deluser --help 随时查看完整参数列表。

问题三：字符编码导致的终端输出乱码

现象：通过 Windows 宿主机终端 SSH 远程连接到虚拟机时，部分中文输出显示为乱码，而虚拟机本地 GNOME 终端显示正常。

原因分析：宿主机与虚拟机终端使用的字符编码不一致。实验中已检查确认虚拟机端 LANG=zh_CN.UTF-8，问题出在 SSH 客户端的终端编码设置上。

解决方法：将终端字符编码统一设置为 UTF-8。在 VS Code 终端中，依次检查：设置 → 搜索 terminal.integrated.defaultProfile，或在终端配置文件中添加 "encoding": "utf-8"。也可通过环境变量 export LANG=en_US.UTF-8 临时切换为英文界面避免乱码。

7、心得体会

技术层面

通过本次实验，我对 Linux 操作系统有了从”图形界面使用者”到”命令行管理者”的认知转变。实验让我掌握了三个核心技能：

  1. 命令行操作能力：从基本的文件操作（ls、cd、cp、rm）到系统资源监控（free、df、top），命令行提供了一种高效、精确的与操作系统交互的方式。相比图形界面，命令行在批量操作、自动化脚本和远程管理方面具有显著优势。

  2. 用户与权限管理：理解了 Linux 多用户体系的设计哲学——通过 UID/GID 和权限位（rwx），实现了精细化的资源访问控制。root 与普通用户的分权设计体现了安全的最小权限原则。

  3. 环境变量机制：环境变量是 Linux 系统的”全局配置神经”，掌握 PATH 的设置和 ~/.bashrc 的编辑方法，是每个开发者必备的技能。系统级（/etc/profile.d/）与用户级（~/.bashrc）的双层配置体系，既保证了系统默认行为的统一，又给用户留出了充分的个性化空间。
